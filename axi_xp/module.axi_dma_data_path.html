Module axi_dma_data_path

Data path for the AXI DMA. This modules handles the R/W channel of the AXI protocol. Module gets read stream, realigns data and emits a write stream. AXI-like valid/ready handshaking is used to communicate with the rest of the backend.

Parameters

DataWidth: int

Data width of the AXI bus

BufferDepth: int

Number of elements the realignment buffer can hold. To achieve full performance a depth of 3 is minimally required.

StrbWidth: int

OffsetWidth: int

Ports

clk_i: input logic

Clock

rst_ni: input logic

Asynchronous reset, active low

r_dp_valid_i: input logic

Handshake: read side of data path is presented with a valid request

r_dp_ready_o: output logic

Handshake: read side of data path is ready to accept new requests

w_dp_valid_i: input logic

Handshake: write side of data path is presented with a valid request

w_dp_ready_o: output logic

Handshake: write side of data path is ready to accept new requests

data_path_idle_o: output logic

High if the data path is idle

r_data_i: input logic [DataWidth-1:0]

Read data from the AXI bus

r_valid_i: input logic

Valid signal of the AXI r channel

r_last_i: input logic

Last signal of the AXI r channel

r_resp_i: input logic [ 1:0]

Response signal of the AXI r channel

r_ready_o: output logic

Ready signal of the AXI r channel

r_tailer_i: input logic [OffsetWidth-1:0]

number of bytes the end of the read transfer is short to reach a Bus-aligned boundary

r_offset_i: input logic [OffsetWidth-1:0]

number of bytes the read transfers starts after a Bus-aligned boundary

r_shift_i: input logic [OffsetWidth-1:0]

The amount the read data has to be shifted to write-align it

w_data_o: output logic [DataWidth-1:0]

Write data of the AXI bus

w_strb_o: output logic [StrbWidth-1:0]

Write strobe of the AXI bus

w_valid_o: output logic

Valid signal of the AXI w channel

w_last_o: output logic

Last signal of the AXI w channel

w_ready_i: input logic

Ready signal of the AXI w channel

w_offset_i: input logic [OffsetWidth-1:0]

number of bytes the write transfers starts after a Bus-aligned boundary

w_tailer_i: input logic [OffsetWidth-1:0]

number of bytes the end of the write transfer is short to reach a Bus-aligned boundary

w_num_beats_i: input logic [ 7:0]

Number of beats requested by this transfer

w_is_single_i: input logic

True if the transfer only consists of a single beat
