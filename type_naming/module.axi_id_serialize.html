Module axi_id_serialize

Reduce AXI IDs by serializing transactions when necessary.

This module is designed to remap a wide ID space to an arbitrarily narrow ID space. If necessary, this module maps two different IDs at its subordinate port to the same ID at its manager port, thereby constraining the order of those transactions and in this sense serializing them. If the independence of IDs needs to be retained at the cost of a wider ID space at the manager port, use axi_id_remap instead.

This module contains one axi_serializer per manager port ID (given by the MgrPortMaxUniqIds parameter).

Parameters

SbrPortIdWidth: int unsigned

ID width of the AXI4+ATOP subordinate port

SbrPortMaxTxns: int unsigned

Maximum number of transactions that can be in flight at the subordinate port. Reads and writes are counted separately (except for ATOPs, which count as both read and write).

MgrPortIdWidth: int unsigned

ID width of the AXI4+ATOP manager port

MgrPortMaxUniqIds: int unsigned

Maximum number of different IDs that can be in flight at the manager port. Reads and writes are counted separately (except for ATOPs, which count as both read and write).

The maximum value of this parameter is 2**MgrPortIdWidth.

MgrPortMaxTxnsPerId: int unsigned

Maximum number of in-flight transactions with the same ID at the manager port.

AddrWidth: int unsigned

Address width of both AXI4+ATOP ports

DataWidth: int unsigned

Data width of both AXI4+ATOP ports

UserWidth: int unsigned

User width of both AXI4+ATOP ports

AtopSupport: bit

Enable support for AXI4+ATOP atomics

sbr_port_axi_req_t: type

Request struct type of the AXI4+ATOP subordinate port

sbr_port_axi_rsp_t: type

Response struct type of the AXI4+ATOP subordinate port

mgr_port_axi_req_t: type

Request struct type of the AXI4+ATOP manager port

mgr_port_axi_rsp_t: type

Response struct type of the AXI4+ATOP manager port

SelectWidth: int unsigned

Number of bits of the subordinate port ID that determine the mapping to the manager port ID

MuxIdWidth: int unsigned

ID width after the multiplexer
